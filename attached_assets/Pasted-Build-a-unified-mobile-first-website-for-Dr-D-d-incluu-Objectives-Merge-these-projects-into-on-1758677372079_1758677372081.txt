Brand system

Auto-extract tokens

Fetch CSS and assets from dr-dede.com and incluu.us.

Parse primary/secondary color variables or compute dominant palette from SVG/PNGs.

Produce brand.tokens.ts:

  dede: { primary: "<#hex>", secondary: "<#hex>", accent: "<#hex>" },
  incluu: { primary: "<#hex>", secondary: "<#hex>", accent: "<#hex>" },

Tailwind config: map tokens to theme.extend.colors.brand.*.

Build steps

Create repo and Next.js app:

npx create-next-app@latest apps/web --ts --eslint --tailwind --app
pnpm i -D @types/node


Add packages:

pnpm add class-variance-authority tailwind-merge lucide-react framer-motion next-seo contentlayer @next/mdx


shadcn/ui:

pnpm dlx shadcn-ui@latest init -d packages/ui
pnpm dlx shadcn-ui@latest add button card navigation-menu dialog accordion


Ingest legacy projects

Pull code/assets from:

Replit: dr-dede-static, dr-dede, incluu, incluu-static, rar-landing-clean

GitHub: dedet2/dr-dede-tedx-homepage

Place images in apps/web/public/brand/...

Convert legacy sections into React Server Components under apps/web/app/(site)/_components.

Brand token extraction task

Script: scan public/brand SVGs + CSS for hex codes; output brand.tokens.ts.

Fallback to manual palette if extraction fails.

Contentlayer

Create packages/content/posts for long-form; add MDX loader; migrate TEDx page copy and messaging.

CI/CD

GitHub Actions: build + preview.

Deploy to Replit or Vercel. Set NEXT_PUBLIC_BASE_URL and analytics env.
